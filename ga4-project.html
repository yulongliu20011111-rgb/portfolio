Case Study: E-commerce Funnel Analysis

GA4 Hypothesis Generation: The Checkout Bottleneck

This project is a reconstructed analytical scenario based on a simulated e-commerce dataset inspired by large-scale platforms. The goal was to practice identifying conversion bottlenecks using Google Analytics 4 (GA4) data and translating them into actionable hypotheses for optimization.

1 The Business Challenge

The challenge was to understand why increasing site traffic did not translate into higher overall purchase rates. Using a simulated dataset reflecting common e-commerce user journeys, I focused on identifying where users dropped off in the checkout funnel, specifically observing behavioral differences across device categories.

2 Core Analysis & Technical Implementation

Funnel Construction

I built a funnel analysis using standard GA4-style e-commerce event data flowing through: add_to_cart → begin_checkout → add_shipping_info → purchase.

The Discovery

The data revealed a severe disparity. Mobile users exhibited a significantly higher drop-off rate (~60–65%) at the "add_shipping_info" step compared to desktop users (~20–25%).

3 Insight & Business Action

This pattern suggests potential friction in the mobile checkout experience, such as form complexity, loading issues, or UX inconsistencies. However, given the reconstructed nature of the dataset, this should be treated as a hypothesis rather than a confirmed causal relationship.

Proposed Optimization Strategy

  • A/B Test Mobile Checkout: Run an experiment with a simplified address input flow (e.g., automated zip code lookup).
  • Performance Audit: Optimize page loading speeds and script execution specifically for mobile users on the shipping page.
  • Friction Reduction: Remove or collapse non-essential, optional fields in the shipping data step to reduce cognitive load.

Estimated Impact: Potential improvement in mobile conversion rate, pending validation through controlled experiments.

Analyst Reflection

This project highlighted how easy it is to over-interpret quantitative funnel data without qualitative validation. While GA4 tells us where users are dropping off, it does not definitively tell us why. In a real-world setting, I would cross-reference this quantitative data with qualitative tools (e.g., session recordings, heatmaps, or error logs) to confirm the root cause before requesting engineering resources for a fix.
