Case Study: E-commerce Growth

GA4 Funnel Optimization: Solving the Purchase Bottleneck

Analyzing how to identify and optimise conversion bottlenecks on high-traffic e-commerce platforms using Google Analytics 4 (GA4), using Amazon as an example, to turn traffic advantages into actual revenue.

1 Data Acquisition: Beyond Basic Tracking

When dealing with platforms on the scale of Amazon, traditional web scraping is relatively inefficient due to dynamic rendering and anti-scraping measures. Therefore, the following two approaches are often used to obtain in-depth data:

  • 01

    BigQuery Export (Raw Data)

    By leveraging the native integration between GA4 and BigQuery, we can access daily raw event stream data. This allows us to bypass GA4’s front-end thresholding and perform more precise user ID association analysis.

  • 02

    Python API Extraction

    Use the Python library google-analytics-data to automatically extract conversion data for specific dimensions (such as Device Category and Landing Page) in preparation for large-scale data cleaning.

2 The Bottleneck: High Traffic, Low Purchase

The Symptom

Weekly website sessions increased by 20%, but the purchase rate decreased by 15%. Using GA4’s Path Exploration, we discovered that a significant amount of traffic was being lost after users added items to their carts.

The Discovery

Using Funnel Exploration, we found that the drop-off rate for mobile users at the “Shipping Selection” step is as high as 65%, while for desktop users it is only 20%.
